## O que foi feito

### 1. Nova skill: `/validar-importacao-ghe`
- **Arquivo**: `.claude/skills/validar-importacao-ghe/SKILL.md`
- Pipeline: Python/openpyxl extrai pares (setor, cargo, qtd) da planilha → SQL extrai catálogo +
  headcount do banco → diff Python com `norm()` para matching case-insensitive → relatório tabular.
- Cobre 3 dimensões: catálogo (empresa_setores/empresa_funcoes), headcount (empresa_headcount), QTD.

### 2. Bug: empresa_headcount ficava órfão ao limpar catálogo
- **Arquivo modificado**: `pseg-admin-questionario.html` — função `salvarGHE()`
- **Causa raiz**: `salvarGHE()` só deletava `empresa_headcount` quando havia um novo import
  (`_gheImportHeadcount.length > 0`). Limpar o catálogo sem reimportar deixava os registros
  antigos intactos.
- **Fix**: adicionado `else if (setorNomes.length === 0 && todosCargos.length === 0)` que força
  DELETE em `empresa_headcount` mesmo sem novo import.
- **Sintoma que revelou o bug**: tela de empresas exibia "207 func." com "0 setores / 0 cargos".
- **Limpeza em PROD**: 335 linhas órfãs deletadas via SQL (172 ELEVA IT + 163 Assa Abloy
  Shared Services).

### 3. Bug: Salvaguarda 1 de `salvarGHE()` bloqueava limpeza intencional
- **Arquivo modificado**: `pseg-admin-questionario.html` — função `salvarGHE()`
- **Causa raiz**: código retornava com `toast` de erro quando catálogo estava vazio, sem dar
  saída ao usuário que queria limpar intencionalmente (ex: trocar de planilha, resetar empresa
  de teste).
- **Fix**: hard-block substituído por `confirm()` com texto explicativo. Usuário pode cancelar
  (preserva dados) ou confirmar (apaga).
- **Decisão**: `confirm()` preserva a proteção contra acidentes sem bloquear fluxos legítimos.

### 4. Ordenação alfabética de setores e cargos no formulário
- **Arquivo modificado**: `pseg-forms.html`
- Setores ordenados por `localeCompare('pt-BR')` em `carregarGHEEmpresa()`.
- Cargos ordenados por `localeCompare('pt-BR')` em `popularMetaFuncao()`.
- "Outro" sempre inserido por último (lógica separada, fora do sort).

### 5. Investigação: dados ausentes no formulário público (Udinese)
- **Causa raiz**: importação feita *antes* do fix cargo→setor 1:N (sessão anterior). Admin
  mostrava setores corretos mas `empresa_funcoes` não tinha `setor_id` preenchido para todos os
  vínculos — o bug descartava vínculos quando o mesmo cargo aparecia em múltiplos setores.
- **Confirmação importante**: formulário lê banco a cada load (`carregarGHEEmpresa()`), sem cache.
  Reimportar pelo admin reflete imediatamente no link existente — não é necessário gerar novo link.

## Decisões tomadas e porquê

| Decisão | Porquê |
|---------|--------|
| `confirm()` em vez de hard-block na Salvaguarda 1 | Hard-block bloqueava fluxos legítimos (limpar empresa de teste, trocar planilha). `confirm()` mantém a proteção sem ser irreversível. |
| DELETE de headcount ao limpar catálogo intencionalmente | Headcount sem catálogo é dado inútil e enganoso. Manter seria pior — causaria confusão operacional com números fantasma. |
| Reimport em vez de fix manual de SQL para GAPs | Reimportar confirma que o pipeline atual está correto para futuras importações. Editar SQL diretamente mascara bugs residuais. |
| `localeCompare('pt-BR')` para ordenação | Handles accented characters corretamente (ex: "Ações" antes de "Belo"). `sort()` padrão coloca acentuados no final em JS. |
